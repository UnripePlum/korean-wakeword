# Worker Issue Contract

Worker issues are the durable wakeword generation queue.

This repository is public. Issue bodies must not contain Threads access tokens, raw platform IDs, private requester metadata, or any data that is not safe to publish.

## Title

```text
요청: <wakeword>
```

Example:

```text
요청: 자비스
```

The title is human-readable. Automation must use the JSON payload in the body as the source of truth.

## Labels

Required lifecycle labels:

- `queued`
- `ready-to-train`
- `training`
- `published`
- `failed`
- `rejected`

Optional labels:

- `needs-review`
- `retryable`
- `metrics-warning`
- `experimental`

## Body Format

The issue body must contain one fenced JSON block.

```json
{
  "schema_version": 1,
  "source": "request_collector",
  "collector_request_id": "req_20260602_a1b2c3d4",
  "raw_phrase": "자비스",
  "normalized_phrase": "자비스",
  "language": "ko",
  "artifact_slug": "ko_jabeuseu_a1b2c3d4",
  "distribution_repo": "UnripePlum/korean-wakeword",
  "created_at": "2026-06-02T00:00:00Z"
}
```

## Required Fields

- `schema_version`: integer, currently `1`.
- `source`: must be `request_collector`.
- `collector_request_id`: public-safe idempotency key generated by the private collector.
- `raw_phrase`: original requested phrase after collector trimming.
- `normalized_phrase`: normalized Korean phrase.
- `language`: must be `ko`.
- `artifact_slug`: deterministic ASCII file stem.
- `distribution_repo`: must be `UnripePlum/korean-wakeword`.
- `created_at`: UTC ISO 8601 timestamp.

## Private Data Boundary

The private request collector keeps:

- raw Threads post ID;
- raw Threads reply ID;
- requester account ID;
- follower or allowlist decision details;
- status mirror state.

The public worker issue keeps only:

- public-safe request ID;
- requested wakeword phrase;
- artifact slug;
- generation status;
- public artifact URLs;
- non-secret metrics.

## Validation Rules

Automation must reject the payload if:

- JSON is missing or malformed;
- `schema_version` is unsupported;
- `source` is not `request_collector`;
- `language` is not `ko`;
- title does not start with `요청:`;
- `artifact_slug` does not match the locally recomputed slug;
- `collector_request_id` was already processed;
- `distribution_repo` is not the expected public repository.

## Idempotency

`collector_request_id` is the public worker idempotency key.

If the collector sees the same Threads reply again, it should update the existing worker issue instead of creating a new one.

If the worker sees an issue already labeled `published`, it must exit without retraining.
